# 机器学习基础概念

> 《动手学深度学习》第一章核心概念整理

## TL;DR

**机器学习** 是让计算机从数据中自动学习规律的方法：

- **核心流程**: `数据 → 模型 → 调参数 → 让预测越来越准`
- **基本概念**: 数据集、样本、特征、维数、标签
- **学习本质**: 反复调整参数，最小化损失函数
- **关键假设**: i.i.d.（独立同分布）
- **核心问题**: 过拟合（背答案 vs 学规律）
- **深度学习 vs 传统方法**: 各有适用场景，不是替代关系

---

## 1. 什么是机器学习

我们不直接编写规则，而是让程序从数据中自动学习：

```text
传统编程:  规则 + 数据 → 结果
机器学习:  数据 + 结果 → 规则（模型自动学出来的）
```

学习过程：

1. 从一个随机初始化参数的模型开始
2. 获取一些数据样本
3. 调整参数，使模型在这些样本中表现得更好
4. 重复 2 和 3，直到表现令人满意

---

## 2. 数据集、样本、特征

三者是一个**三层包含关系**：

```text
数据集（Dataset）= 整张表
┌──────────────────────────────────────────────────────┐
│  样本（Sample）    面积    卧室数    到市中心距离    价格（标签）│
│  ─────────────   ──────  ───────  ──────────────  ────────── │
│  房子 1           120㎡     3        2km           300万      │
│  房子 2           80㎡      2        5km           150万      │
│  房子 3           200㎡     4        1km           500万      │
└──────────────────────────────────────────────────────┘
                    ←────── 特征（Features）──────→   ← 标签 →
```

| 概念 | 是什么 | 对应表格中的 |
|------|--------|------------|
| **数据集** | 所有样本的集合 | 整张表 |
| **样本** | 一个具体的观测对象 | 一**行** |
| **特征** | 描述样本的属性 | 一**列**（或多列） |
| **标签** | 要预测的目标值 | 特殊的一列 |

### 图像数据的特征

图像的**每一个像素值就是一个特征**：

```text
一张 200×200 彩色照片:

  200 × 200 × 3(RGB) = 120,000 个数值
                        ↑
                   120,000 个特征

展开成特征向量:
  [r₁, g₁, b₁, r₂, g₂, b₂, ..., r₄₀₀₀₀, g₄₀₀₀₀, b₄₀₀₀₀]
```

## 3. 维数（Dimensionality）

**维数 = 一个样本有几个特征 = 特征向量的长度。**

```text
1 个特征 → 一维：数轴上的一个点
2 个特征 → 二维：平面上的一个点
3 个特征 → 三维：空间中的一个点
N 个特征 → N 维：N 维空间中的一个点（画不出来，但算得出来）
```

N 维空间不需要能"看到"，它只是说"描述一个东西需要 N 个数字"。计算机不需要"看到"空间，只需要算数字。

比如两张图片之间的"距离"：

```text
3维空间：
  距离 = √((x₁-x₂)² + (y₁-y₂)² + (z₁-z₂)²)        3项

120,000维空间：
  距离 = √((x₁-x₂)² + (x₁-x₂)² + ... + (x₁-x₂)²)  120,000项

  公式结构完全一样
```

### 固定长度 vs 不固定长度

| 类型 | 示例 | 维数 |
|------|------|------|
| **固定长度** | 房价数据（每套房都有面积、卧室数、距离） | 统一维数，方便处理 |
| **不固定长度** | 文本（"好极了" vs "这个产品非常好用推荐购买"） | 长度不一，需要特殊处理 |

深度学习的一个主要优势就是可以处理不同长度的数据。

---

## 4. i.i.d. 假设

大多时候，我们假设样本遵循**独立同分布**（independently and identically distributed, i.i.d.）：

- **独立**: 样本之间互不影响
- **同分布**: 所有样本来自同一个概率分布

### 语音数据违反 i.i.d.

语音流数据几乎在所有层面都违反 i.i.d.：

| 层面 | 违反情况 |
|------|---------|
| **独立性** | 相邻帧之间存在极强的时序依赖（声道是惯性系统、音素按顺序排列） |
| **同分布** | 说话人不同、环境不同、设备不同，分布会漂移 |

工程实践中通过分帧、切片、数据增强和 shuffle 在**训练样本层面近似 i.i.d.**，同时用 [[序列模型]] 在推理时显式建模时序依赖。

## 5. 目标函数与损失函数

### 目标函数 = 给参数打分的公式

它回答一个问题：当前模型的参数好不好？

```text
模型：房价 = w × 面积

w = 1   → 预测 100，实际 200，打分 = (100-200)² = 10000  差
w = 1.5 → 预测 150，实际 200，打分 = (150-200)² = 2500   好一些
w = 2   → 预测 200，实际 200，打分 = (200-200)² = 0      完美
```

"学习"就是沿着损失曲线往最低点走。

### 平方误差（Squared Error）

最常见的损失函数：**(预测值 - 真实值)²**

为什么用平方而不是直接算差值？

| 需求 | 平方误差 | 直接加误差 | 绝对值误差 |
|------|---------|-----------|-----------|
| 正负不抵消 | 是 | 否（+20 和 -20 相加为 0） | 是 |
| 大错误重罚 | 是（偏差 20 的惩罚远大于偏差 10） | 否 | 否（线性增长） |
| 光滑可求导 | 是（处处光滑） | 否 | 否（零点有尖角） |

### 替代目标（Surrogate Objective）

有些真正想优化的目标没法直接优化，需要换一个能算的来代替：

```text
分类任务中：

  真正目标（错误率）：猜对/猜错，阶梯形，没法求导 → 无法指导学习
  替代目标（交叉熵）：光滑曲线，处处可求导 → 可以学习

  交叉熵越小 → 正确类别的概率越高 → 猜对越多 → 错误率越低
  替代目标 ↓  ≈  真正目标 ↓   （方向一致，所以能代替）
```

### 交叉熵（Cross-Entropy）

分类任务中最常用的损失函数：**-log(模型给正确答案的概率)**

```text
一张猫图，模型输出 P(猫) = 0.7

交叉熵 = -log(0.7) = 0.36
```

| 模型给正确答案的概率 | 交叉熵 | 含义 |
|-------------------|--------|------|
| 0.01 | 4.61 | 猜得离谱，重罚 |
| 0.3 | 1.20 | 不太确定 |
| 0.7 | 0.36 | 比较自信 |
| 0.99 | 0.01 | 非常确定，几乎无惩罚 |

交叉熵比错误率更"细腻"——能区分"勉强猜对"（P=0.51）和"自信猜对"（P=0.99）。

### log 函数

log 回答的问题是"底数要乘几次才等于这个数"：

```text
2 × 2 × 2 = 8  →  log₂(8) = 3（"2 乘 3 次等于 8"）
```

ML 中用 log 的原因：

- **压缩大数字**: 概率连乘容易变成极小数（0.0000003），取 log 变成正常的加法（-15.0）
- **-log(p) 的形状天然适合当损失**: p 接近 0 惩罚极大，p 接近 1 惩罚为 0
- **光滑可求导**: log(x) 的导数 = 1/x，简单好算

### 模型为什么输出概率而不是直接给答案？

```text
模型内部流程：

  图片输入 → 一顿矩阵运算 → 原始数字 [猫: 2.5, 狗: 1.2]
                                        ↓
                              Softmax（把数字转成概率）
                                        ↓
                              概率 [P(猫)=0.79, P(狗)=0.21]
                                        ↓
                              取最大的 → "是猫"
```

原因：概率是连续数值，可以一点一点调整来逐步学习。"猫"或"狗"是离散标签，没法微调。

## 6. "学习"的过程

**"学"就是反复调整参数，让预测和真实答案之间的差距越来越小。**

```text
第 1 步：猜一个参数值
第 2 步：用数据算一下差多少（损失）
第 3 步：朝"差更少"的方向调一调参数
重复 2 和 3，直到差得足够小 → 学完了
```

### 训练集与测试集

| 数据集 | 作用 | 类比 |
|--------|------|------|
| **训练集** | 用于调整模型参数 | 模拟考试 |
| **测试集** | 用于评估模型在新数据上的表现 | 真正的期末考试 |

训练集上表现好不代表测试集上也好，就像模拟考考得好不一定期末考也好。

## 7. 过拟合（Overfitting）

**过拟合 = 模型把数据中的随机噪声也当成了规律。**

```text
真实规律:  一条直线（房价 = 2 × 面积）
训练数据:  直线上的几个点 + 一些随机波动（噪声）

正常学习:  找到了那条直线（忽略噪声） → 新数据也能预测
过拟合:    画了一条扭曲的曲线穿过每个点  → 训练集完美，新数据崩溃
```

### 为什么会过拟合？

参数太多 + 数据太少 → 满足数据的答案太多 → 模型随便挑了一个（大概率不是真正的规律）

```text
3 个数据点，1 个参数 → 解几乎唯一 → 被迫找到规律
3 个数据点，6 个参数 → 无穷多解 → 随机选到一个扭曲的曲线
```

### 怎么解决

| 方法 | 原理 |
|------|------|
| 加数据 | 数据越多，噪声影响越小，规律越明显 |
| 简化模型 | 减少参数，逼模型只学大趋势 |
| 正则化 | 人为惩罚过大的参数，限制模型的"自由度" |
| 早停 | 训练到"差不多"就停，别继续钻研噪声 |

## 8. 深度学习 vs 传统机器学习

**不是替代关系，各有各的地盘。**

| 场景 | 谁更合适 | 原因 |
|------|---------|------|
| 100 条数据预测房价 | 传统（线性回归、决策树） | 数据太少，深度学习过拟合 |
| 表格数据（Excel） | 传统（XGBoost） | 特征含义明确，不需要自动提取 |
| 图像分类 | 深度学习（CNN） | 像素级特征人工提不好 |
| 语音识别 | 深度学习（Transformer） | 变长序列、复杂模式 |

### 为什么深度学习能处理变长数据？

**参数共享**：同一组参数在输入的每个位置上复用。

```text
传统方法：每个位置一套专属参数 → 长度变了参数就对不上 → 必须固定长度
深度学习：同一组参数在每个位置复用 → 输入变长只是多复用几次 → 任意长度
```

以 RNN 为例，只有一个"员工"，从第一个字读到最后一个字：

```text
每一步做的事（永远同一个公式）：
  新的理解 = f(上一步的理解, 当前这个字)

3 个字 → 执行 3 次 f
10000 个字 → 执行 10000 次 f
同一个 f，参数不变
```

### 为什么深度学习在小数据上反而更差？

参数太多 + 数据太少 = 过拟合

```text
100 条数据，3 个特征

线性回归：学 3 个参数 → 100 条数据绰绰有余
深度学习：学几万个参数 → 100 条数据远远不够
```

## 9. 三种学习方式

```text
监督学习：  有标准答案   "这张是猫，这张是狗"     ← 用得最多
无监督学习：没有答案     "这里有一堆图片，你自己找规律"
强化学习：  只有奖惩     "做对了给奖励，做错了扣分，自己摸索"
```

监督学习依然是主力——只要有标准答案，它就是最简单高效的方法。现代大模型（如 ChatGPT）是先用监督学习打基础（90%），再用强化学习（RLHF）微调回答风格（10%），两者配合而非替代。

详细的监督学习任务类型见 [[监督学习任务]]。
