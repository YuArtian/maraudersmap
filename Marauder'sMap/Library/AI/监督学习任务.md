# 监督学习任务

> 监督学习 = 给标准答案，让模型照着学。这里介绍四种常见任务类型。

## TL;DR

**监督学习**擅长在"给定输入特征"的情况下预测标签，常见任务：

- **回归**: 输出一个数值（房价、温度）
- **分类**: 输出一个类别（猫/狗）
- **搜索**: 给候选结果打分并排序
- **推荐**: 猜用户可能喜欢什么并排序
- **核心区别**: 回归和分类输出单个答案，搜索和推荐输出一个排好序的列表

---

## 1. 回归（Regression）

**回答"有多少"的问题，输出一个连续数值。**

```text
输入：面积 120㎡ → 输出：350万（一个具体的数字）
输入：今日气象数据 → 输出：28.5°C
```

| 特点 | 说明 |
| --- | --- |
| 输出 | 连续数值（可以是 350、351、349.5，任意数字） |
| 损失函数 | 平方误差 `(预测 - 真实)²` |
| 例子 | 房价预测、温度预测、销量预测 |

---

## 2. 分类（Classification）

**回答"是哪个"的问题，输出一个类别。**

```text
输入：一张图片 → 输出：猫（一个类别）
输入：一封邮件 → 输出：垃圾邮件 / 正常邮件
```

### 模型不直接输出类别，而是输出概率

```text
图片输入 → 模型计算 → 原始数字 [猫: 2.5, 狗: 1.2]
                              ↓
                    Softmax（转成概率）
                              ↓
                    P(猫) = 0.79, P(狗) = 0.21
                              ↓
                    取最大的 → "是猫"
```

为什么输出概率而不是直接给答案？

- 概率是连续数值，可以一点点调整来逐步学习
- "猫"或"狗"是离散标签，没法微调
- 交叉熵损失需要概率才能计算

| 特点 | 说明 |
| --- | --- |
| 输出 | 每个类别的概率，取最大的 |
| 损失函数 | 交叉熵 `-log(正确类别的概率)` |
| 例子 | 图像识别、垃圾邮件、手写数字识别 |

### 回归 vs 分类

| | 回归 | 分类 |
| --- | --- | --- |
| 问题 | 有**多少**？ | 是**哪个**？ |
| 输出 | 连续数值（350万） | 离散类别（猫/狗） |
| 损失函数 | 平方误差 | 交叉熵 |

---

## 3. 搜索（Search）

**给候选结果打相关性分数，按分数排序。**

```text
用户搜索："如何学 Python"

  网页 A "Python 入门教程"      → 相关性: 0.95
  网页 B "Python 官方文档"      → 相关性: 0.90
  网页 C "Java 和 Python 对比"  → 相关性: 0.60
  网页 D "今日天气预报"          → 相关性: 0.01

  按分数排序：A → B → C → D
```

**顺序很重要**——用户只看前几条，最相关的排到第 10 页等于没找到。

### PageRank

谷歌早期算法：被越多网页链接的网页越重要（像投票），现代搜索引擎进一步结合用户行为（点击、停留时间）来打分。

---

## 4. 推荐系统（Recommendation）

**猜用户可能喜欢什么，给"用户-物品"配对打分并排序。**

```text
系统猜你喜欢什么：

  (你, 商品A) → 喜欢概率: 0.90
  (你, 商品B) → 喜欢概率: 0.15
  (你, 商品C) → 喜欢概率: 0.75

  排序后推给你：A → C → B
```

### 搜索 vs 推荐

| | 搜索 | 推荐 |
| --- | --- | --- |
| 触发方式 | 用户主动输入关键词 | 系统自动生成 |
| 输入信息 | 明确的查询词 | 用户历史行为（浏览、点击、购买） |
| 目标 | 找和查询词最**相关**的 | 找用户最**可能喜欢**的 |
| 例子 | 百度、谷歌 | 抖音推视频、淘宝推商品、网易云推歌 |

底层都是**打分 + 排序**，区别在于输入——搜索靠查询词，推荐靠用户历史。
