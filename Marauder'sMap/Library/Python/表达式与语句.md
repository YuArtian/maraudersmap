# 表达式与语句

## TL;DR

**表达式（Expression）能算出一个值，语句（Statement）执行一个动作。** 核心要点：

- 判断方法：能放在 `=` 右边的就是表达式
- 三元表达式 `a if 条件 else b` 是 Python 版的 `条件 ? a : b`
- 理解这个区别，就能理解列表生成式里 if/else 的规则

---

## 1. 表达式 vs 语句

| | 表达式（Expression） | 语句（Statement） |
|---|---|---|
| 定义 | 能算出一个**值**的代码片段 | 执行一个**动作**，不产生值 |
| 判断方法 | 能放在 `=` 右边 | 不能放在 `=` 右边 |

```python
# 表达式 -- 每个都能算出一个值
1 + 2              # → 3
x * x              # → 某个数
"hello".upper()    # → "HELLO"
x > 5              # → True 或 False
x if x > 0 else -x # → 某个数
len([1, 2, 3])     # → 3

# 语句 -- 执行动作，没有值
x = 10             # 赋值语句
if x > 0:          # if 语句
    print(x)
for i in range(3): # for 语句
    pass
import os          # import 语句
```

最简单的验证：

```python
result = 1 + 2                    # OK，表达式
result = x if x > 0 else -x      # OK，表达式
result = len([1, 2])              # OK，表达式
result = for i in range(3): pass  # SyntaxError! 语句不能放这里
```

## 2. 三元表达式

Python 的条件表达式（三元表达式）：

```python
a if 条件 else b
```

| Python | JavaScript |
|--------|-----------|
| `a if 条件 else b` | `条件 ? a : b` |

```python
# Python
status = "偶数" if x % 2 == 0 else "奇数"

# JavaScript（等价）
// const status = x % 2 === 0 ? "偶数" : "奇数"
```

三元表达式是**表达式**，所以能出现在任何需要值的地方：赋值右边、函数参数、列表生成式的 for 前面等。

---

## 3. 为什么这个概念重要

理解了表达式和语句的区别，很多语法规则都不用死记：

**列表生成式**：`for` 左边是表达式位，必须算出值；`for` 右边是子句位，只接受 `if` 过滤。

```python
[表达式  for x in ...  if 过滤条件]
```

- `x if c else y` 能放在 for 前面，因为它是表达式（有值）
- `x if c` 不能，因为它不是完整的表达式（算不出值）
- for 后面的 `if` 是子句语法，不是表达式，所以不能带 `else`

**f-string 的 `{}` 里**：只能写表达式，不能写语句。

```python
f"result: {x + 1}"           # OK，表达式
f"result: {x = 10}"          # SyntaxError! 赋值是语句
```

---

## 4. Python vs JS 对比

| 概念 | Python | JavaScript |
|------|--------|-----------|
| 条件表达式 | `a if c else b` | `c ? a : b` |
| 条件语句 | `if c: ... else: ...` | `if (c) {...} else {...}` |
| 赋值 | 语句（`x = 1`） | 表达式（`x = 1` 有返回值） |
| 函数调用 | 表达式 | 表达式 |
| lambda | 表达式（`lambda x: x+1`） | 表达式（`x => x+1`） |

注意：JS 中赋值是表达式（`x = 1` 返回 `1`），所以能写 `if (x = getValue())`。Python 中赋值是语句，不能这么写（Python 3.8+ 引入了海象运算符 `:=` 来解决这个问题）。
