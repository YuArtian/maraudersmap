# 面向对象编程

## TL;DR

**Python 的类支持基于类的继承（非原型链），并且支持多继承。** 核心要点：

- 属性查找顺序：实例属性 → 类属性 → 父类属性，和 JS 原型链行为一致
- 多继承用 MRO（Method Resolution Order）解决方法查找顺序
- `super().__init__()` 确保父类初始化逻辑被执行
- `__slots__` 限制实例可绑定的属性，同时节省内存
- MixIn 是利用多继承"混入"功能的设计模式

---

## 1. 实例属性 vs 类属性

```python
class Student(object):
    name = 'Student'       # 类属性

s = Student()
s.name                     # 'Student' -- 实例没有，找到类属性
s.name = 'Michael'         # 给实例绑定同名属性（遮蔽类属性）
s.name                     # 'Michael'
del s.name                 # 删掉实例属性
s.name                     # 'Student' -- 类属性又可见了
```

查找顺序和 JS 原型链完全一样：

```text
Python:  实例属性 → 类属性 → 父类属性 → object
JS:      实例属性 → prototype → 父级prototype → Object.prototype
```

## 2. 继承

Python 是基于类的继承，比 JS 原型链更直接：

```python
class Animal(object):
    def run(self):
        print('Animal is running')

class Dog(Animal):         # Dog 继承 Animal
    def run(self):         # 重写（Override）
        print('Dog is running')
```

### 调用父类方法

用 `super()` 调用父类的方法，最常见的是调用 `__init__`：

```python
class Dog(Animal):
    def __init__(self, name):
        super().__init__()   # 确保父类初始化逻辑执行
        self.name = name
```

不调用 `super().__init__()` 的话，父类 `__init__` 里设置的属性就丢了：

```python
class Animal:
    def __init__(self):
        self.alive = True

class Dog(Animal):
    def __init__(self, name):
        # 没有 super().__init__()
        self.name = name

d = Dog('旺财')
d.alive  # AttributeError! 父类初始化没执行
```

JS 的 `constructor` 里不调用 `super()` 会直接报错，Python 不报错但会静默丢属性。

| | Python | JavaScript |
|---|---|---|
| 继承语法 | `class Dog(Animal):` | `class Dog extends Animal` |
| 调用父类 | `super().__init__()` | `super()` |
| 不调用的后果 | 静默丢属性 | 直接报错 |

## 3. 多继承与 MRO

Python 支持多继承，JS 不支持：

```python
class Flyable:
    def fly(self):
        print('Flying')

class Swimmable:
    def swim(self):
        print('Swimming')

class Duck(Animal, Flyable, Swimmable):
    pass

d = Duck()
d.run()    # 从 Animal 继承
d.fly()    # 从 Flyable 继承
d.swim()   # 从 Swimmable 继承
```

### MRO（Method Resolution Order）

多继承时同名方法用哪个？Python 按 MRO 顺序查找（从左到右）：

```python
class A:
    def say(self): print('A')

class B(A):
    def say(self): print('B')

class C(A):
    def say(self): print('C')

class D(B, C):   # B 在 C 前面
    pass

D().say()  # 'B' -- 先找到 B

# 查看完整查找顺序
print(D.__mro__)
# (D, B, C, A, object)
```

### 菱形问题

```text
      A
     / \
    B   C
     \ /
      D
```

B 和 C 都重写了 A 的方法，D 该用谁的？Python 用 MRO 解决（按声明顺序从左到右），JS 选择从设计上避免 -- 不支持多继承。

## 4. MixIn

MixIn 是利用多继承"混入"功能的**设计模式**，不是语法特性：

```python
class FlyableMixIn:
    def fly(self):
        print('Flying')

class SwimmableMixIn:
    def swim(self):
        print('Swimming')

# 像搭积木一样组合
class Duck(Animal, FlyableMixIn, SwimmableMixIn):
    pass
```

```text
Animal          ← 主继承链（是什么）
FlyableMixIn    ← 混入功能（能做什么）
SwimmableMixIn  ← 混入功能（能做什么）
```

### MixIn 的约定

| 规则 | 原因 |
|------|------|
| 类名以 `MixIn` 结尾 | 一眼看出它是混入 |
| 不定义 `__init__` | 不干扰主类初始化 |
| 只提供方法，不存状态 | 避免属性冲突 |
| 功能单一 | 一个 MixIn 只做一件事 |

### 标准库实例

```python
# socketserver 模块
class ThreadingTCPServer(ThreadingMixIn, TCPServer): pass
class ForkingUDPServer(ForkingMixIn, UDPServer): pass
```

### JS 的等价写法

JS 没有多继承，用函数嵌套模拟：

```javascript
const FlyableMixin = (Base) => class extends Base {
    fly() { console.log('Flying') }
}

class Duck extends FlyableMixin(Animal) {}
```

## 5. `__slots__`

限制实例能绑定哪些属性：

```python
class Student:
    __slots__ = ('name', 'age')  # 只允许这两个

s = Student()
s.name = 'Bob'    # OK
s.score = 99      # AttributeError!
```

两个作用：

| 作用 | 说明 |
|------|------|
| 限制属性 | 防止拼写错误，访问不存在的属性立刻报错 |
| 节省内存 | 不再为每个实例创建 `__dict__`，大量实例时差别明显 |

注意：`__slots__` 只对当前类有效，**不继承到子类**。子类也要限制就得自己定义 `__slots__`。

JS 的类似功能是 `Object.seal()`：

```javascript
const s = { name: 'Bob', age: 20 }
Object.seal(s)
s.score = 99  // 静默失败（严格模式报错）
```

## 6. `__init__` 是可选的

`__init__` 不是必须的，等同于 JS 的 `constructor`：

```python
# 不需要初始化就不写
class Student(object):
    name = 'Student'

s = Student()  # 不传参

# 需要初始化才写
class Student(object):
    def __init__(self, name):
        self.name = name

s = Student('Bob')  # 必须传 name
```
