# 字符编码

## TL;DR

计算机只能处理数字，文本必须转成数字。演化路线：**ASCII → 各国编码（乱码）→ Unicode（统一编号表）→ UTF-8（省空间的存储方式）**。现在基本一切都是 UTF-8。

---

## 1. 演化路线

```text
ASCII（美国）
  ↓ 只有英文，各国不够用
GB2312（中国）/ Shift_JIS（日本）/ Euc-kr（韩国）...
  ↓ 各国编码互相冲突 → 乱码
Unicode（全球统一编号表）
  ↓ 全用 2+ 字节存英文太浪费
UTF-8（变长编码，按需 1~4 字节）
```

类比：
1. **ASCII** — 美国人造了个电话本，只收录 127 个英文字符
2. **各国编码** — 中日韩各自造电话本，互相打电话对不上号（乱码）
3. **Unicode** — 联合国搞了个全球统一电话本，每个字符一个唯一编号
4. **UTF-8** — 聪明的压缩方案，英文 1 字节，中文 3 字节，按需伸缩

---

## 2. ASCII

用 **1 个字节**（8 bit），只用了 7 bit（0~127），收录：

- 大小写英文字母、数字、标点符号、控制字符
- 例：`A` = 65，`z` = 122，`0` = 48

1 个字节最多表示 256 个字符，英文够了，中文（几万个字）远远不够。

### bit 和字节的关系

| 字节数 | bit 数 | 能表示的最大整数 |
|--------|--------|-----------------|
| 1 | 8 | 255（2⁸ - 1） |
| 2 | 16 | 65535（2¹⁶ - 1） |
| 4 | 32 | 4294967295（2³² - 1） |

减 1 是因为 0 也占一个位置。

## 3. Unicode

Unicode 是一张**"字符 → 编号"的对照表**，给每个字符分配一个唯一编号（叫**码点**）：

```text
码点           字符
U+0041        A
U+0061        a
U+0030        0
U+4E2D        中
U+3042        あ
U+1F600       😀
```

### 码点的写法

`U+` 前缀 + 十六进制数字。用十六进制是因为**和字节对齐**：

```text
1 个十六进制位 = 4 bit
2 个十六进制位 = 8 bit = 1 字节
```

每个十六进制位和 4 个 bit 一一对应，可以直接互相转换：

```text
二进制：    0100 1110  0010 1101
十六进制：  4    E     2    D     ← 每位对应 4 个 bit，一一对齐
十进制：    20013                  ← 完全看不出二进制长什么样
```

对照表：

```text
0 = 0000    4 = 0100    8 = 1000    C = 1100
1 = 0001    5 = 0101    9 = 1001    D = 1101
2 = 0010    6 = 0110    A = 1010    E = 1110
3 = 0011    7 = 0111    B = 1011    F = 1111
```

看到 `4E` 可以直接拼出 `0100 1110`，而十进制 `78` 没法直接知道二进制是什么。

### 十六进制计算方式

和十进制一样，每位乘以权重再相加（底数从 10 变成 16）：

```text
十进制 325 = 3×10² + 2×10¹ + 5×10⁰ = 325
十六进制 4E2 = 4×16² + E×16¹ + 2×16⁰ = 1024 + 224 + 2 = 1250
```

（A=10, B=11, C=12, D=13, E=14, F=15）

Python 中可直接验证：`0x4E2` → `1250`

### Unicode 的范围

Unicode 不只是 2 个字节。目前收录超过 14 万个字符，编号从 0 到 0x10FFFF：

| 范围 | 内容 |
|------|------|
| U+0000 ~ U+007F | ASCII（英文、数字） |
| U+0080 ~ U+07FF | 拉丁文、希腊文、阿拉伯文等 |
| U+4E00 ~ U+9FFF | 常用中文 |
| U+10000 ~ U+10FFFF | emoji、生僻字、古文字等 |

```python
>>> ord('A')
65              # 1 字节范围
>>> ord('中')
20013           # 2 字节范围
>>> ord('𠮷')    # 生僻字
131127          # 超出 2 字节，需要更多
>>> ord('😀')
128512          # emoji，也超出 2 字节
```

### Unicode 只是编号表

Unicode **不规定用几个字节存储**。怎么存是编码方式（UTF-8/16/32）的事：

- **Unicode** = 一张表，给每个字符一个编号
- **UTF-8 / UTF-16 / UTF-32** = 把编号存进电脑的具体方式

## 4. UTF-8

把 Unicode 码点编码成 **1~4 个字节**的变长编码，英文省空间，兼容 ASCII：

| 字符 | Unicode 码点 | UTF-8 编码 | 字节数 |
|------|-------------|-----------|--------|
| A | U+0041 | `01000001` | 1 |
| 中 | U+4E2D | `11100100 10111000 10101101` | 3 |
| 😀 | U+1F600 | 4 字节 | 4 |

### 与其他编码方式对比

| 编码方式 | 每个字符占用 | 特点 |
|----------|------------|------|
| UTF-8 | 1~4 字节（变长） | 最省空间，兼容 ASCII，用得最多 |
| UTF-16 | 2 或 4 字节 | Windows/Java 内部用 |
| UTF-32 | 固定 4 字节 | 最浪费空间，但定位快 |

同一个字符在不同编码下占用不同：

| 字符 | UTF-8 | UTF-16 | UTF-32 |
|------|-------|--------|--------|
| A | 1 字节 | 2 字节 | 4 字节 |
| 中 | 3 字节 | 2 字节 | 4 字节 |
| 😀 | 4 字节 | 4 字节 | 4 字节 |

### UTF-8 的额外好处

ASCII 编码是 UTF-8 的子集——ASCII 文件天然就是合法的 UTF-8 文件。大量只支持 ASCII 的历史遗留软件可以在 UTF-8 下继续工作。

---

## 5. Python 3 默认 UTF-8

老教程常见这两行开头声明：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
```

- 第一行告诉 Linux/macOS 这是 Python 可执行程序（Windows 忽略）
- 第二行告诉 Python 解释器按 UTF-8 读取源代码

**Python 3 已经不需要写了**——默认编码就是 UTF-8。Python 2 时代默认 ASCII，遇到中文会报错，所以必须声明。现在的编辑器（VS Code 等）保存文件也默认 UTF-8，两边都不用操心。

---

## 6. 现代系统的工作方式

```text
内存中：Unicode（统一编号，方便处理）
存硬盘 / 传网络：UTF-8（省空间）
```

Python 中用 `\u` 和 `\U` 可以直接写 Unicode 码点：

```python
>>> '\u0041'
'A'
>>> '\u4e2d'
'中'
>>> '\U0001F600'
'😀'
```
