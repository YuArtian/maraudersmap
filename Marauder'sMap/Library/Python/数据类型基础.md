# Python 数据类型基础

## TL;DR

Python 区分 `int` 和 `float` 两种数字类型。整数**无限精度**，浮点数有限制（64 位 IEEE 754）。布尔值大写开头（`True`/`False`），空值用 `None`。与 JS 的关键区别：JS 的 `Number` 全是浮点数。

---

## 1. 整数 (int)

Python 整数**没有大小限制**，数字小时占内存少，大了自动扩展：

```python
>>> 2 ** 10
1024
>>> 2 ** 1000
# 正常输出一个巨大的数，不会溢出
```

### 其他语言有限制

大多数语言用固定大小的内存存整数：

| 字节数 | bit 数 | 最大值 | 计算方式 |
|--------|--------|--------|----------|
| 1 | 8 | 255 | 2⁸ - 1 |
| 2 | 16 | 65535 | 2¹⁶ - 1 |
| 4 | 32 | 4294967295（约 42 亿） | 2³² - 1 |

Java 的 32 位有符号整数（1 位符号 + 31 位数值）：-2³¹ 到 2³¹-1（约 ±21 亿），超过就溢出。

JS 的 `Number` 全是 64 位浮点数，整数安全范围是 -2⁵³+1 到 2⁵³-1（约 ±9000 万亿），超过运算就不精确。后来加了 `BigInt`（加 `n` 后缀）来解决。

---

## 2. 浮点数 (float)

Python 浮点数是 **64 位 IEEE 754**，和 JS 的 `Number` 底层一样：

- 最大约 1.8 × 10³⁰⁸，超过变成 `inf`（无穷大）
- 精度 53 位，可能有四舍五入误差

```python
>>> 1.8e308
1.8e+308
>>> 1.8e309
inf
```

### int vs float 对比

| | `int` | `float` |
|---|---|---|
| 大小限制 | 无限制，自动扩展 | 最大约 1.8e308 |
| 超出后 | 正常算，只是变慢 | 变成 `inf` |
| 精度 | 完全精确 | 53 位精度，可能有误差 |
| 有无小数点 | `1` | `1.0` |

```python
>>> type(1)
<class 'int'>
>>> type(1.0)
<class 'float'>
>>> 1 == 1.0     # 值相等（Python 自动转换后比较）
True
>>> 1 is 1.0     # 不是同一个对象
False
```

### 整数除法是精确的

| 运算符 | 含义 | 例子 | 结果 |
|--------|------|------|------|
| `//` | 整数除法（取整） | `10 // 3` | `3`（int，精确） |
| `/` | 普通除法 | `10 / 3` | `3.3333...`（float，可能不精确） |

只要在整数世界里运算（加减乘和 `//`），结果永远精确。

### 大整数和浮点数怎么比较

Python **不会**无脑把大 `int` 转成 `float`（那样会丢精度），而是用专门的比较逻辑：

```python
>>> 2**53 + 1 == float(2**53 + 1)
False    # Python 能发现 float 丢了精度
```

### 与 JS 的关键区别

| | Python | JavaScript |
|---|---|---|
| 数字类型 | `int` 和 `float` 分开 | 只有 `Number`（全是 float） |
| `1` 和 `1.0` | 不同类型 | 完全一样 |
| 整数能无限大 | 能（int 专门实现） | 不能（受 float 精度限制，除非用 BigInt） |

## 3. 布尔值

Python 大写开头：

| | Python | JavaScript |
|---|---|---|
| 真 | `True` | `true` |
| 假 | `False` | `false` |
| 空 | `None` | `null` |

Python 里写 `true` 会报错 `NameError`。

## 4. None

表示"什么都没有"，用于：

**函数没有返回值：**

```python
def say_hi():
    print("hi")

result = say_hi()
print(result)   # None
```

**函数可能找不到结果：**

```python
def find_user(name):
    if name in database:
        return user
    return None          # 没找到
```

返回 `0` 或 `""` 会引起混淆（它们是有意义的值），`None` 明确表示"没有"。

**变量先占位：**

```python
result = None
if condition:
    result = 100
```

**参数默认值：**

```python
def greet(name=None):
    if name is None:
        print("你好，陌生人")
    else:
        print(f"你好，{name}")
```

与 JS 对比：Python 只有一个 `None`，JS 要区分 `null` 和 `undefined`。

## 5. 字符串与三引号

多行字符串可以用三引号，避免 `\n` 难读：

```python
# 用 \n 写，难读
s = '第一行\n第二行\n第三行'

# 用三引号，直观
s = '''第一行
第二行
第三行'''
```

`'''...'''` 和 `"""..."""` 都行，效果完全一样。
