# Python 环境管理

## TL;DR

Python 环境管理有两条独立路线：**Conda（Anaconda/Miniconda）** 和 **pyenv + pip + venv**。一个"环境"本质上就是**一个文件夹**，里面有独立的 Python 和库。选一条路线用就行，学深度学习推荐 Conda。

---

## 1. 两条路线

```text
路线 A：pyenv + pip + venv
路线 B：Conda（Anaconda / Miniconda）
```

它们是完全独立的两套体系，没有依赖关系。

---

## 2. Anaconda

一个 **Python 发行版**（打包好的安装包），面向数据科学和机器学习。

类比：
- **Python 官方** = 裸机电脑，什么软件都没有，需要什么自己装
- **Anaconda** = 预装好常用软件的电脑，开箱即用

预装内容：
- Python 本身
- 几百个常用数据科学库（numpy、pandas、matplotlib、scikit-learn、jupyter 等）
- **Conda**（包管理器 + 环境管理器）

**Miniconda** 是轻量版——只装 Python + Conda，不预装几百个库。

---

## 3. Conda

Anaconda 里最核心的**命令行工具**，干两件事：装包 + 管环境。

### 3.1 装包

```bash
conda install numpy
conda install pytorch
conda remove numpy
conda list              # 看当前环境装了什么
conda search xxx        # 搜索可用的包
```

和 pip 的区别：pip 只能装 Python 包，Conda 能装非 Python 的依赖（C 库、CUDA 工具包等），所以装 PyTorch 这类复杂的库更省心。

### 3.2 管环境

```bash
conda create -n d2l python=3.13   # 创建环境
conda activate d2l                 # 进入环境
conda deactivate                   # 退出，回到 base
conda env list                     # 列出所有环境
conda remove -n d2l --all          # 删掉整个环境
```

终端开头的括号是当前环境名：

```text
(base) yuartian@... %        ← 默认环境
(d2l)  yuartian@... %        ← 切到了 d2l 环境
```

### 3.3 环境是什么

一个环境就是**一个文件夹**，里面装着一套独立的 Python 和库：

```bash
conda env list
# base     /Users/yuartian/anaconda3
# d2l      /Users/yuartian/anaconda3/envs/d2l
```

文件夹结构：

```text
/Users/yuartian/anaconda3/envs/d2l/
├── bin/
│   ├── python          ← 这个环境自己的 Python
│   └── pip             ← 这个环境自己的 pip
└── lib/
    └── python3.13/
        └── site-packages/
            ├── numpy/   ← 这个环境装的库
            ├── torch/
            └── ...
```

`conda activate d2l` 做的事情就是**改 PATH**，让 `python` 命令指向 `d2l` 文件夹里的 Python，而不是 `base` 里的。

### 3.4 为什么要隔离环境

项目 A 需要 `numpy 1.x`，项目 B 需要 `numpy 2.x`，放两个文件夹就不会打架。就像手机上装 app——每个项目是一个"手机"，装自己需要的 app，互不干扰。

### 3.5 常用命令速查

| 命令 | 作用 |
|------|------|
| `conda --version` | 查看 Conda 版本 |
| `conda env list` | 列出所有环境 |
| `conda list` | 当前环境的所有包 |
| `conda list -n 环境名 python` | 查看某环境的 Python 版本 |
| `conda install xxx` | 装包 |
| `conda update xxx` | 更新包 |
| `conda search xxx` | 搜索可用的包 |
| `conda create -n 名字 python=3.x` | 创建新环境 |
| `conda activate 名字` | 切换环境 |
| `conda deactivate` | 退出当前环境 |
| `conda remove -n 名字 --all` | 删掉整个环境 |

查看当前环境 Python 版本直接用 `python --version`。

## 4. pyenv

**只管 Python 版本**的工具，不管库、不管环境：

```bash
pyenv install 3.13    # 装 3.13
pyenv install 3.14    # 装 3.14
pyenv global 3.13     # 全局用 3.13
pyenv local 3.14      # 这个项目目录用 3.14
```

需要搭配 pip（装库）和 venv（虚拟环境）使用。

### pyenv vs Conda 对比

| | pyenv | Anaconda/Conda |
|---|---|---|
| 管 Python 版本 | 是（专门干这个） | 是（顺带能管） |
| 管第三方库 | 不管（交给 pip） | 管（conda install） |
| 管虚拟环境 | 不管（交给 venv） | 管（conda create） |
| 预装科学库 | 不预装 | 预装几百个 |
| 体积 | 很小 | 很大（几个 GB） |

类比：
- **pyenv** = 遥控器，专门切电视频道（Python 版本）
- **Anaconda** = 一整套家庭影院系统，全包了

---

## 5. 为什么终端里有那么多 python 命令

同时装了 Conda 和 pyenv，Python 版本就会很多：

| 命令 | 来源 |
|------|------|
| `python` | Conda base 环境（日常用这个） |
| `python3` | 系统自带 |
| `python3.13`、`python3.14` | pyenv 装的 |
| `python3-config` 等 | C 扩展编译工具，不用管 |
| `pythonw`、`python.app` | macOS GUI 相关，不用管 |
| `python-build` | pyenv 的内部工具 |

日常用 Conda 那条路就行，pyenv 装的可以无视。
